# Dynamic Service Pages - Setup Guide

## ✅ What's Fixed
The booking form dropdown now **automatically includes ALL service pages** from `/src/pages/services/` folder. No manual updates needed!

## 🔄 How It Works

1. **Auto-detection**: The system scans your `/src/pages/services/` folder and detects all `.jsx` and `.js` files
2. **Auto-generation**: Creates a list file at `/src/data/autoGeneratedServices.js`
3. **Dynamic dropdown**: The booking form reads this list and shows all services

## 📝 When You Add a New Service Page

Whenever you create a new service page, follow these simple steps:

### Step 1: Create your service page
```
/src/pages/services/your-new-service-dubai.jsx
```

### Step 2: Update the services list
Run this command in terminal:
```bash
npm run generate:services
```

### Step 3: That's it! 
Your new service will automatically appear in the booking form dropdown! 🎉

## 🚀 Quick Commands

```bash
# Generate services list (run after adding new pages)
npm run generate:services

# Start development server
npm run dev

# Build for production (auto-generates during build)
npm run build
```

## 💡 Pro Tips

1. **Always run** `npm run generate:services` after creating a new service page
2. **Don't edit** `/src/data/autoGeneratedServices.js` manually - it's auto-generated
3. **File naming**: Use kebab-case like `ayurvedic-diet-weight-loss-dubai.jsx`
4. **The dropdown** automatically formats names nicely (removes "dubai", "treatment", etc.)

## ❓ Troubleshooting

**Service not showing in dropdown?**
1. Make sure the file is in `/src/pages/services/`
2. File should end with `.jsx` or `.js`
3. Run `npm run generate:services`
4. Restart your dev server: `npm run dev`

**Dropdown still empty?**
Check console for errors and make sure `/src/data/autoGeneratedServices.js` exists.
